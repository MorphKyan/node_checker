# JSON 文件存储与 API 风险评分重构

## 总体设计

- 删除“总分”概念；节点只保留一个 `risk_score: number | null`，值为启用 API 的加权聚合风险分，越低风险越小。
- TTFB 不参与风险计算，仅用于判断真实代理连接是否有效和展示延迟。
- TTFB 成功即连接有效；高风险、地区不符、TCP Ping/路由/IPv6 失败均不改变有效性。
- 所有 API 无有效结果时，连接仍有效，`risk_score=null`，界面显示“风险未知”。
- 项目运行时彻底移除 SQLite。选择 JSON，因为无需新增依赖、适合嵌套配置与列表，并与现有前后端数据格式一致。

## JSON 存储架构

统一使用可配置的 `DATA_DIR`，默认布局：

```text
data/
  config.json
  results/{subscription_id}.json
  jobs/{job_id}.json
  probe-cache/{cache_key}.json
```

- `config.json` 保存版本号、运行设置、出口 IP 端点、API 站点、订阅定义和 Sing-box 模板。
- API Key 以明文保存在本地 JSON，但所有读取接口只返回 `api_key_configured`；日志、结果和异常不得泄漏 Key 或完整鉴权 URL。
- 检测结果、任务状态和探测缓存分别使用独立 JSON 文件，避免单个大文件在并发探测时频繁整体重写。
- 所有写入通过同目录临时文件、刷新落盘和 `os.replace` 原子替换；进程内使用细粒度锁保护配置、任务和单个结果写入。
- 探测缓存文件包含过期时间，读取时惰性删除过期或损坏条目；缓存键加入节点指纹和完整探测配置签名。
- 删除订阅时同步删除对应结果；修改订阅 URL 时清除旧结果并让旧任务失效；服务重启时把遗留的 queued/running 任务标记为失败。
- 删除 SQLite Store、建表迁移、DB 环境变量和相关测试代码。已有 `.sqlite3` 文件不迁移、不删除，也不再读取。
- Docker 只需持久化 `/app/data`，改用 `DATA_DIR=/app/data`，不再设置 `API_DB_PATH`、`CACHE_DB_PATH` 等数据库路径。
- 维持当前单 Uvicorn worker 部署模型；不支持多个服务进程同时写同一数据目录。

## API 站点与探测流程

- 使用“供应商适配器注册表 + JSON 站点配置”，支持增删、启停、排序和配置：
  - 唯一站点 ID
  - 节点表格列名
  - 供应商类型
  - HTTP(S) URL 模板
  - API Key
  - 聚合权重
  - 启用状态与顺序
- URL 模板必须包含 `{ip}`，可选 `{key}`。适配器统一处理请求、状态分类、响应归一化、地区/ASN 和风险证据提取。
- 首次生成 `config.json` 时预置六种现有供应商；启用 ipwho.is、ipapi.is、proxycheck.io、IP2Location，其余默认关闭，并保留当前默认权重。
- 出口 IP 使用独立、可配置的代理内查询端点，不作为评分来源或节点表格列；支持 JSON `ip` 字段和纯文本 IP。
- 每次刷新开始时冻结 API 配置快照；运行中修改配置不影响当前任务，下一次刷新才生效。
- 聚合风险只纳入成功且能提供显式风险分或可归一化风险标签的 API：
  - 优先使用供应商返回的数值风险分。
  - 没有数值时，根据统一风险标签映射估算。
  - 按站点配置权重计算 0–100 加权平均。
  - 没有任何可用风险证据时返回 `null`。
- TTFB 至少一个代理请求样本成功即视为真实连接成功；TCP Ping、路由和 IPv6 继续作为辅助信息并行采集。
- 删除旧的 `NodeAnalyzer` 评分扣分、最低总分、高风险淘汰、地区不符淘汰、供应商专用结果字段和硬编码调用链。

## 接口与前端调整

- 新增 API 站点接口：
  - `GET /api-sites`
  - `GET /api-sites/providers`
  - `POST /api-sites`
  - `PATCH /api-sites/{id}`
  - `DELETE /api-sites/{id}`
  - `PUT /api-sites/order`
- Key 更新时留空表示保持不变，通过显式 `clear_api_key` 清除。
- `ApiVerdict` 增加 `site_id` 和成功、超时、限流、错误、无数据等状态。
- `SubscriptionResults` 增加当次运行的 API 站点列快照；历史结果使用保存时的列名和顺序。
- 从节点结果中删除 `total_score` 和 `score_details`；`risk_score` 改为可空。
- 删除所有 `min_score` 参数和最低分筛选，替换为 `max_risk`：
  - 已知风险必须小于等于阈值。
  - 未知风险默认不通过 `max_risk` 筛选。
- 导出排序改为：已知风险升序、未知风险最后、TTFB 升序、已测速速度降序。
- 增强节点名称中的“XX分”改为“风险 XX”；无 API 结果显示“风险未知”。
- 新增左侧“API 站点”标签页，包含基础出口 IP 端点、站点 CRUD、排序、启停、列名、URL、Key 和权重设置。
- 节点表格按结果快照动态生成 API 列；每格显示网络标签、风险标签和该 API 风险分，失败时显示具体状态。
- 总览中的平均总分和评分分布改为平均风险、风险区间分布及风险未知数量。
- `valid_only` 只过滤 TTFB 连接失败节点；风险筛选必须显式使用 `max_risk`。
- 普通刷新、首次刷新默认 `speedtest_limit=0`。保留“测速刷新”入口，由用户明确输入每区域测速数量并可选择强制探测。
- 测速结果使用 `speedtest_status` 和可空速度，区分未测速、测速失败和成功；无测速数据时不绘制全零排行榜。

- 测试 JSON 配置默认生成、校验、原子更新、损坏文件处理、并发任务写入和服务重启恢复。
- 测试订阅更新/删除与结果文件联动、任务状态更新、过期缓存清理及配置签名缓存失效。
- 验证项目运行路径不再导入 `sqlite3`，不创建或读取任何 SQLite 文件。
- 覆盖六类供应商适配器、URL/Key 渲染、风险归一化、权重聚合和错误状态。
- 验证 TTFB 不影响风险数值，只影响连接有效性；API 全失败返回 `risk_score=null`。
- 验证高风险、地区不符及辅助探测失败不会把真实连接成功的节点判为无效。
- 覆盖 API 站点页、Key 遮罩、动态列、未知风险、`max_risk` 筛选、风险排序和默认无测速流程。
